## What the Code Does

### `main.py`

A single-file Pygame application with the following structure:

**Initialization**
- `pygame.init()` — starts all Pygame subsystems
- `pygame.display.set_mode((800, 600))` — creates an 800x600 window
- `pygame.display.set_caption("Pong")` — sets the window title
- `pygame.time.Clock()` — creates a clock for framerate control

**The Game Loop**
```
while running:
    handle input      ← check for quit, keypresses
    clock.tick(60)    ← cap loop at 60 frames per second
    screen.fill()     ← wipe the canvas black
    draw everything   ← render paddles, ball, score
    display.flip()    ← swap back buffer to screen
```

This loop is the heartbeat of every game ever made. It runs 60 times per second until the player quits.

**Rendering**
- `screen.fill((0, 0, 0))` — paints the screen black every frame
- `pygame.draw.rect()` — draws paddles as white rectangles
- `pygame.display.flip()` — flips the back buffer to the screen (prevents flickering)

## What Was Learned Building This

### Lesson 1: Python Fundamentals
- **Variables** — Python uses dynamic typing. `running = True` creates and assigns in one step. No type declaration needed.
- **Functions** — `def main():` defines a function. `main()` calls it. Defining is writing the recipe. Calling is cooking it.
- **While loops** — run until a condition is false. The game loop is a while loop because you don't know how long the player will play.
- **f-strings** — `f"Tick: {tick}"` for clean string formatting with variables inline.

### Lesson 2: The Game Loop Concept
- A game loop has three jobs every frame: check input, update state, draw to screen.
- It can never block. Instead of waiting for input, it checks whether input happened.
- Two separate while loops don't work — the second never runs because the first never ends.

### Lesson 3: Pygame Basics
- `clock.tick(60)` — without this, the loop runs as fast as the CPU allows. With it, consistent 60fps on any machine.
- `screen.fill()` — without this, old frames leave trails on screen.
- `pygame.display.flip()` — Pygame draws to a hidden back buffer. `flip()` swaps it to the front. This prevents flickering.
- `pygame.event.get()` — non-blocking input. Checks what happened this frame without waiting.

### Lesson 4: Tooling
- `pyenv` — manages multiple Python versions on one machine. Different projects need different versions.
- `pip show pygame` — checks if a library is installed and which Python it's on.
- `git init`, `git add`, `git commit`, `git push` — the commit workflow.
- `gh repo create` — creates a GitHub repo from the terminal without touching a browser.

### Lesson 5: State
- `running' was originally a boolean
- game_state gives more granular control of game states
